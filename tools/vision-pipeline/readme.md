# Vision Pipeline

Offline tool for generating animal-vision-filtered exhibit images used in the
InteractiveZooScreens React Native app (Santa Barbara Zoo).

These are **not** app source files. This folder lives outside the Expo project
and is run manually by a developer whenever exhibit images need to be created
or updated. The outputs (filtered JPEGs) are what get committed to the app.

---

## Asset folder structure

All animal vision images live under a single `animalVisions/` folder, split into two subfolders:

```
assets/
└── images/
    ├── backgrounds/              ← other screen backgrounds (parallax, etc.) — unrelated
    └── animalVisions/
        ├── originals/            ← source exhibit images (input to the pipeline)
        │   └── fiji.jpeg
        └── filtered/             ← pipeline output, never edit manually
            └── fiji-fiji_iguana.jpeg
```

**originals/** is where you put the raw exhibit photo before running the pipeline.
**filtered/** is where the pipeline writes its output. Never put files here by hand.

## Tool folder structure

```
tools/vision-pipeline/
├── README.md                   ← you are here
├── requirements.txt            ← Python dependencies (numpy, Pillow)
├── animal_vision.py            ← core vision simulation engine (do not edit lightly)
└── generate_exhibit_images.py  ← CLI tool you actually run
```

---

## One-time setup

You need Python 3.9+ and pip.

```powershell
cd tools/vision-pipeline
pip install -r requirements.txt
```

That's it. No servers, no Expo, no Node.

## Generating images

Run from inside the `tools/vision-pipeline/` directory.

**Fiji iguana exhibit (the current exhibit):**

```powershell
python generate_exhibit_images.py --input ../../assets/images/animalVisions/originals/fiji.jpeg --output ../../assets/images/animalVisions/filtered/ --animals fiji_iguana
```

This writes one file:

```
assets/images/animalVisions/filtered/fiji-fiji_iguana.jpeg
```

**Preview without writing files:**

```powershell
python generate_exhibit_images.py --input ../../assets/images/animalVisions/originals/fiji.jpeg --output ../../assets/images/animalVisions/filtered/ --animals fiji_iguana --dry-run
```

**Generate all available animal profiles at once:**

```powershell
python generate_exhibit_images.py --input ../../assets/images/animalVisions/originals/fiji.jpeg --output ../../assets/images/animalVisions/filtered/ --animals all
```

## How it works

`animal_vision.py` models each animal's photoreceptor cone sensitivities using
the Govardovskii A1 visual pigment template (a standard biophysics model). It
builds a 3x3 color-mixing matrix from the difference between the animal's cone
peak wavelengths and human cone peaks, applies it in gamma-correct linear light,
then adds an oil-droplet saturation/exposure effect to simulate the pigmented
droplets found in many reptile and bird retinas.

The math is grounded in real cone sensitivity data. The output is an
approximation — a best-effort translation of what that animal's visual system
would encode — not a literal photograph from the animal's perspective.

## Adding a new animal profile

Open `animal_vision.py` and add an entry to `ANIMAL_PROFILES`:

```python
ANIMAL_PROFILES = {
    # ... existing profiles ...

    "your_animal": {
        "animal_cone_peaks_nm": [S_nm, M_nm, L_nm],   # short, mid, long cone peaks in nm
        "oil_droplet_saturation": 1.0,                 # >1 = more vivid, 1.0 = no change
        "oil_droplet_exposure": 1.0,                   # <1 = dimmer, 1.0 = no change
        "description": "Brief description for logging.",
    },
}
```

Then run the generator:

```powershell
python generate_exhibit_images.py --input ../../assets/images/animalVisions/originals/<your-image>.jpg --output ../../assets/images/animalVisions/filtered/ --animals your_animal
```

**Where do cone peak values come from?**
Published literature. Good starting points:

- Loew & Govardovskii (2001) for reptiles
- Hart (2001) for birds
- Jacobs (2009) for mammals
  The existing profiles in `animal_vision.py` include citations in their description fields.

## Adding a new exhibit image

1. Put the source exhibit image in `assets/images/animalVisions/originals/`.
2. Run the generator pointing `--input` at the new image.
3. Commit both the source image and the filtered output(s).
4. Update `VISION_IMAGES` in `AnimalVisionScreen.tsx` (see comment in that file).

---

## Connecting the output to the app

In `AnimalVisionScreen.tsx`, the `VISION_IMAGES` object maps mode keys to asset
requires. After generating a new filtered image, update it like this:

```typescript
// Generated by tools/vision-pipeline/generate_exhibit_images.py
// To regenerate: see tools/vision-pipeline/README.md
const VISION_IMAGES = {
  human: require("../assets/images/animalVisions/originals/fiji.jpeg"),
  fijiBandedIguana: require("../assets/images/animalVisions/filtered/fiji-fiji_iguana.jpeg"),
};
```

The mode toggle buttons in the screen just call `setActiveVision("fijiBandedIguana")`
or `setActiveVision("human")` — no processing happens at runtime.

## Output naming convention

```
<input-stem>-<profile-key><extension>
```

Examples:
| Input (originals/) | Profile | Output (filtered/) |
|---|---|---|
| `fiji.jpeg` | `fiji_iguana` | `fiji-fiji_iguana.jpeg` |
| `savanna-bg.jpg` | `dog` | `savanna-bg-dog.jpg` |
| `ocean-exhibit.png` | `bird` | `ocean-exhibit-bird.png` |

## Committing generated images

Commit both the original and filtered images to the repo. They are treated as
static assets just like any other image in `assets/`. Do not gitignore them —
the app cannot bundle an asset it cannot find at build time.

If the source exhibit image changes, regenerate and recommit all filtered
variants for that exhibit.
